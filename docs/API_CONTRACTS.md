# API Contracts

## Node API

### `GET /api/health`

Returns gateway health and persistence mode.

### `POST /api/auth/guest`

Starts a fresh backend guest session and returns a bearer token for that guest session.

### `POST /api/auth/signup`

Creates an email/password account and returns a bearer token plus the signed-in session.

Request:

```json
{
  "name": "<display-name>",
  "email": "<user-email>",
  "password": "<user-password>"
}
```

### `POST /api/auth/login`

Logs into an existing email/password account and returns a bearer token plus the signed-in session.

### `GET /api/auth/session`

Restores the current backend session from the bearer token in the `Authorization` header.

### `POST /api/auth/logout`

Ends the current backend session.

### `GET /api/sessions/recent`

Returns the latest stored searches for the authenticated user session or the active guest session.

### `POST /api/search`

Request:

```json
{
  "query": "I need to build a resume but I have no design skills",
  "limit": 5,
  "excludeResults": [
    {
      "slug": "canva",
      "name": "Canva",
      "url": "https://www.canva.com/"
    }
  ]
}
```

`Authorization: Bearer <token>` is optional for recommendation responses, but the React app sends it so searches are saved into the right backend guest or user session.

`excludeResults` is optional. The React app sends the current temporary result set when a user clicks
"Search more" so the next batch can replace duplicate matches with fresh recommendations.

Response:

```json
{
  "query": "I need to build a resume but I have no design skills",
  "summary": "For \"I need to build a resume but I have no design skills,\" start with Canva, LinkedIn, and Google Docs so you can move from exploration into your first practical step.",
  "results": [],
  "agentTrace": [],
  "orchestration": {
    "mode": "gpt-4o-mini",
    "milestones": [],
    "iterations": [],
    "assumptions": [],
    "missingRequirements": [],
    "serviceBoundaries": [],
    "architectureNotes": []
  },
  "meta": {
    "cache": "miss",
    "persistenceMode": "memory",
    "agentMode": "gpt-4o-mini"
  }
}
```

### `POST /api/search/network`

Requests a wider recommendation set for the prompt-centered application graph.

Request:

```json
{
  "query": "I need to build a resume but I have no design skills",
  "limit": 50
}
```

Response uses the same shape as `POST /api/search`, but `results` may contain up to 50 distinct applications or sites.

### `GET /api/guides/:slug.doc?query=...`

Streams a Google Docs-compatible starter document generated by the FastAPI service. The document includes
the starter guide content and a checklist for understanding the application.

### `GET /api/guides/:slug.pdf?query=...`

Streams a polished PDF fallback generated by the FastAPI service.

### `GET /api/guides/:slug.html?query=...`

Streams the same starter guide as browser-readable HTML.

## FastAPI Service

### `GET /health`

Returns service health, LLM Brain model, and runtime availability.

### `POST /search`

Receives the user query, a batch limit, and optional excluded result identities. Returns LLM Brain
recommendations plus the agent trace.

### `POST /search/network`

Receives the user query and a limit up to 50. Returns LLM Brain recommendations for the application
network graph plus the agent trace.

### `GET /guides/{slug}.doc?query=...`

Builds a Google Docs-compatible starter document from the cached or reconstructed LLM Brain recommendation.

### `GET /guides/{slug}.pdf?query=...`

Builds a PDF fallback from the cached or reconstructed LLM Brain recommendation.

### `GET /guides/{slug}.html?query=...`

Builds a browser-readable HTML starter guide from the cached or reconstructed LLM Brain recommendation.
